项目名称：基于 ESP32 的多模式通信与远程控制系统
    开发周期：2025.7.1 – 2025.XX
    开发平台：ESP32 (ESP-IDF) + 阿里云 IoT
    开发语言：C、JSON
    使用技术：FreeRTOS、MQTT、TCP/IP、NVS、RMT、GPIO、WS2812、串口协议
项目简介：
    本项目基于 ESP32 实现一套多模式网络通信和智能控制系统，支持 AP/STA Wi-Fi 模式切换、
    双 MQTT 通道通信、TCP 服务、串口配置与协议通信、IO 控制、WS2812 灯效控制等功能，
    适用于 IoT 网关、LED 信息屏、OTA远程升级等场景。
主要功能模块：
    1.Wi-Fi 网络切换（STA/AP 模式）
    AP 模式：ESP32 作为热点，配置静态 IP，运行 TCP Server 接收连接设备数据。
    STA 模式：连接现有 Wi-Fi 网络，使用 DHCP 获取 IP，作为 MQTT 客户端接入云端/本地。
    网络配置由串口发送 JSON 配置动态设置.
    2.双 MQTT 通道
    通道一：连接本地 MQTT Broker 与设备进行局域网通信；
    通道二：连接阿里云物联网平台，主要用于 OTA 远程升级与远程命令；
    动态重连策略，掉线后自动恢复。
    3.串口通信 + RS232 控制
    ESP32 通过 UART 接口（RS232）与下位 LED 显示屏进行通信；
    实现帧头识别、命令解析、数据处理、CRC 校验、屏幕刷新控制。
    4.WS2812 灯带控制（使用 led_strip_rmt 驱动）
    基于 RMT 外设控制 WS2812 RGB 灯带；
    支持红 → 绿 → 蓝 的丝滑颜色过渡；
    5. GPIO 输入/输出与状态保护
    多路 GPIO 控制继电器、蜂鸣器、指示灯等外设；
    多路GPIO输入控制从板显示内容
    6.NVS 配置管理
    使用 NVS 实现设备参数持久化，包括工作模式、IP 配置等；
    系统上电自动加载参数并应用，无需用户重新配置；
    配合串口实现配置读写、恢复出厂设置等功能。
